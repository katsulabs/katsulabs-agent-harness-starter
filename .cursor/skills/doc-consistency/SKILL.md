# 문서 일관성 검증

프로젝트 문서 전체에서 용어, 표기법, 섹션 구조, 역할 경계 설명이 일관되는지 확인하는 스킬입니다.

## 사용 시점

- Contract 에이전트가 용어나 인터페이스 정의를 변경한 후
- 문서를 추가하거나 대폭 수정한 후
- QA 에이전트가 문서 품질 게이트를 검수할 때

## 검증 절차

### 1단계: 역할 용어 일관성 확인

표준 역할 명칭:
- `Main` (오케스트레이터)
- `Contract`
- `Backend`
- `Frontend`
- `QA`

확인 포인트:
- 대소문자가 일관되는가 (예: `backend` vs `Backend`)
- 동의어가 혼용되지 않는가 (예: `Main` vs `Orchestrator`)
- 역할 열거 순서가 일관되는가 (권장: Contract → Backend → Frontend → QA)

### 2단계: 표기법 통일 확인

표준 표기법:

| 올바른 표기 | 잘못된 표기 |
|------------|------------|
| `worktree` | `work tree`, `work-tree` |
| `no-ff` | `no ff`, `noff` |
| `DoD` | `Dod`, `dod`, `DOD` |

### 3단계: 섹션 구조 일관성 확인

`docs/harness/` 내 문서들이 일관된 섹션 구조를 유지하는지 확인합니다.

확인 포인트:
- 제목 체계가 `#` → `##` → `###` 순서를 지키는가
- `#`(h1)은 파일당 1개만 있는가

### 4단계: 에이전트 역할 경계 설명 일관성

각 에이전트의 범위 설명이 `agent-hierarchy.md`와 개별 `.mdc` 규칙 간에 일치하는지 확인합니다.

## 결과 판정

| 결과 | 판정 |
|------|------|
| 용어와 구조 모두 일관 | 통과 |
| 용어 불일치 발견 | 표준 표기로 통일 |
| 역할 경계 설명 불일치 | 기준 문서와 규칙 파일 동기화 |

## 주의사항

- 용어를 변경할 때는 반드시 모든 관련 문서를 함께 갱신합니다.
- 불확실한 용어 정의가 있으면 Contract/Main에 확인 후 진행합니다.
