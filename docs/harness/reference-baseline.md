# 하네스 기본 기준

새 프로젝트 기본 하네스 기준입니다.

## 핵심 기준

- main 보호 + worktree 강제
- no-ff 머지
- 역할 분리 (Main/Contract/Backend/Frontend/QA)
- 테스트 게이트 (backend + frontend)

## 에이전트 행동 원칙 (Karpathy 가이드라인)

모든 에이전트가 준수해야 하는 4가지 행동 원칙이다.
상세 규칙: `.cursor/rules/karpathy-guidelines.mdc`

| 원칙 | 핵심 | 해결하는 문제 |
|------|------|---------------|
| 코딩 전에 생각하기 | 가정하지 않고, 트레이드오프를 표면화 | 잘못된 가정, 숨겨진 혼란 |
| 단순성 우선 | 최소한의 코드, 추측성 구현 금지 | 과도한 복잡성, 비대한 추상화 |
| 수술적 변경 | 필요한 것만 수정, 기존 스타일 준수 | 관련 없는 부수 변경 |
| 목표 기반 실행 | 성공 기준 정의 후 검증까지 반복 | 불명확한 완료 조건 |

사소한 작업(오타 수정, 명확한 한 줄 변경)에는 전체 엄격함을 적용할 필요 없이 판단력을 사용한다.
